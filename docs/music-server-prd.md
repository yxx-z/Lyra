# Lyra — 音乐服务器产品需求文档（PRD）

> 版本：0.1 · 日期：2026-05-31 · 状态：草稿

---

## 1. 背景与目标

### 1.1 问题陈述

视频领域已有 Jellyfin、Emby、飞牛播放器等成熟的自托管方案，支持刮削海报墙、元数据管理和多端播放。但音乐领域缺少同等体验的开源项目：

- **Navidrome**：轻量但刮削能力弱，只读文件内嵌标签，识别率低，中文音乐库尤为明显
- **Beets**：命令行工具，普通用户门槛高，无内置播放器
- **Jellyfin 音乐模块**：附属功能，体验粗糙
- **Plex**：商业化重度，隐私存疑

**用户痛点**：有大量本地音乐文件，想在家庭网络内自托管，同时拥有良好的元数据、封面、歌词和多端播放体验。

### 1.2 产品目标

构建一个开源、自托管的音乐服务器，让拥有本地音乐库的用户能够：

1. 自动识别并刮削准确的元数据（专辑、艺术家、封面、歌词）
2. 在任意设备上流畅播放，支持歌词同步显示
3. 一键部署，无需运维经验

### 1.3 成功指标

- 刮削准确率 ≥ 90%（含中文歌曲）
- 首次部署到可播放 ≤ 5 分钟
- Web UI 可在主流浏览器和移动端正常使用
- 兼容至少 4 款主流 Subsonic 客户端（DSub、Symfonium、Ultrasonic、Feishin）

## 2. 用户画像

### 2.1 主要用户：数字音乐收藏者

**描述**：有数年积累的本地音乐库（1,000～100,000 首），存放在 NAS、家庭服务器或个人电脑上。有一定动手能力，会用 Docker 部署，但不想写代码。

**核心诉求**：
- 音乐库能被"认识"，显示正确的封面和元数据
- 随时随地（家庭局域网 / VPN 穿透）能听
- 歌词同步显示（尤其是 K 歌场景）

**典型场景**：
- 晚上躺床上用手机刷专辑听歌
- 在电脑上工作时开着 Web 播放器当背景音乐
- 发现音乐库里某首歌的专辑封面识别错了，手动修正

### 2.2 次要用户：开发者 / 极客

**描述**：想在已有的 Homelab 里补一个音乐服务，或者对项目本身感兴趣、想贡献代码。

**核心诉求**：
- Subsonic API 兼容，能接入自己的客户端或脚本
- 配置灵活，日志清晰，方便排查问题
- 代码结构清晰，容易二次开发

## 4. 功能需求

### 4.1 文件扫描

- **支持格式**：MP3、FLAC、AAC（m4a）、OGG、OPUS、WAV、AIFF、WMA
- **扫描触发方式**：
  - 启动时自动全量扫描
  - 文件系统事件触发增量扫描（新增 / 修改 / 删除）
  - API / UI 手动触发
- **扫描行为**：
  - 跳过非音频文件
  - 文件路径变更（移动 / 重命名）识别为同一文件，保留播放记录
  - 删除的文件标记为不可用，不自动清除数据库记录（需用户确认清理）

### 4.2 元数据处理

**读取优先级（从高到低）**：
1. 内嵌标签（ID3v2 / Vorbis Comment / iTunes Tag）
2. 基于标签的 MusicBrainz 精确查询（有 MBID）
3. 基于 Title + Artist 的 MusicBrainz 模糊搜索
4. AcoustID 音频指纹识别（标签缺失或低置信度时）
5. 保留原始标签数据（完全识别失败时）

**刮削内容**：
- 曲名、艺术家、专辑艺术家、专辑名、年份、流派、曲目编号、碟片编号
- 专辑封面（本地缓存，支持多尺寸）
- 艺术家简介、艺术家图片
- 发行信息（MusicBrainz Release）

**封面处理**：
- 存储原始分辨率到本地缓存目录
- API 响应时按需缩放（支持 ?size=300 参数）
- 封面优先级：内嵌封面 > 同目录 cover.jpg/folder.jpg > 网易云（中文歌曲）> Last.fm > Spotify（可选，v1.0）

### 4.3 歌词

- **查询来源**（按优先级）：数据库缓存 → 内嵌 LYRICS 标签 → LRCLIB → 网易云音乐 → 纯文本回退（无时间轴）
- **存储格式**：LRC（标准时间轴）+ YRC（逐字，可选）
- **缓存**：结果缓存至数据库，避免重复请求
- **显示**：当前播放行高亮，支持点击跳转

### 4.4 流媒体

- **直传**：格式兼容时直接 HTTP Range 传输
- **转码**：通过 ffmpeg 实时管道转码，支持 MP3 / AAC / OPUS 输出
- **码率选项**：64 / 128 / 192 / 320 kbps（可在请求参数中指定）
- **缓冲策略**：支持 HTTP Range 请求，允许客户端 seek

### 4.5 搜索

- **搜索范围**：曲名、艺术家名、专辑名
- **实现方式**：SQLite FTS5 全文索引
- **中文支持**：集成 jieba 分词或 ICU tokenizer，支持中文分词搜索

### 4.6 播放统计

- 记录每次播放（时间、来源客户端）
- 支持 Subsonic scrobble 接口（同步到 Last.fm 可选）
- 统计数据用于"最常播放"功能

## 5. 非功能需求

### 5.1 性能

| 指标 | 目标 |
|------|------|
| 扫描速度 | ≥ 500 首/秒（仅读标签，无刮削） |
| 刮削速度 | ≥ 10 首/秒（受限于外部 API 频率） |
| 首包延迟 | 直传模式 ≤ 200ms |
| 搜索响应 | ≤ 100ms（10 万首库） |
| 内存占用 | 空闲时 ≤ 100MB |
| 并发流数 | 同时支持 ≥ 10 路流（取决于带宽和转码 CPU） |

### 5.2 可靠性

- 刮削失败不影响已入库歌曲的正常播放
- 外部 API 不可用时降级（使用内嵌标签数据）
- 服务崩溃重启后自动恢复，不丢数据
- SQLite WAL 模式，防止数据库损坏

### 5.3 兼容性

| 类型 | 要求 |
|------|------|
| 操作系统 | Linux（主要）、macOS、Windows |
| CPU 架构 | amd64、arm64（NAS / 树莓派） |
| 浏览器 | Chrome 90+、Firefox 88+、Safari 14+ |
| Subsonic 版本 | API 1.16.1 |

### 5.4 安全性

- 支持配置访问密码（基础 HTTP Auth 或 Token）
- Subsonic API 使用独立密码，与管理密码隔离
- 不向外部服务发送音频内容本身，仅发送元数据用于查询
- 外部 API Key 存储在配置文件中，不写入数据库

### 5.5 可观测性

- 结构化日志输出（JSON 格式可选），支持日志级别配置
- 扫描 / 刮削进度通过 API 实时查询
- 提供 `/health` 端点（用于 Docker 健康检查）

## 6. UI/UX 需求

### 6.1 整体风格

- 深色模式 / 浅色模式，跟随系统设置
- 响应式布局，桌面端和移动端均可正常使用
- 专辑封面优先，视觉风格类似 Apple Music 或 Spotify

### 6.2 核心页面

**专辑墙（首页）**
- 网格布局展示所有专辑封面
- 支持按艺术家 / 年份 / 最近添加排序
- 点击专辑进入曲目列表

**艺术家页**
- 艺术家图片 + 简介
- 该艺术家的专辑列表

**播放器（底栏常驻）**
- 封面缩略图 + 曲名 + 艺术家名
- 进度条（可拖拽 seek）
- 播放控制：上一首 / 播放/暂停 / 下一首
- 音量控制
- 随机 / 循环切换
- 点击封面展开歌词视图

**歌词视图**
- 全屏模糊背景（取专辑封面主色）
- 当前行放大高亮，上下行渐隐
- 逐字高亮（有 YRC 数据时）

**搜索页**
- 输入即搜，不需要按回车
- 结果分区显示：歌曲 / 专辑 / 艺术家

**管理页**
- 库统计（总数、各格式占比）
- 刮削状态列表（待处理 / 成功 / 失败）
- 手动编辑元数据的表单
- 扫描触发按钮 + 进度显示

### 6.3 交互细节

- 刮削置信度不足时，在管理页标记并提供候选项供用户确认，类似 Beets 的交互
- 封面图片加载前显示占位色块（取主色），避免布局跳动
- 切换歌曲时播放器不白屏，过渡流畅

## 7. 数据需求

### 7.1 用户数据

- 服务器不上传任何用户音乐文件或收听数据到开发者服务器
- 刮削请求遵循各 API 服务的隐私政策

### 7.2 数据持久化

- 所有元数据、播放记录存储在本地 SQLite 数据库
- 封面图片缓存到本地目录
- 用户可自行备份整个数据目录

### 7.3 数据迁移

- 数据库版本升级通过迁移脚本自动完成，无需用户干预
- 提供导入 Navidrome 数据库的工具（v1.1 考虑）

## 8. 限制与边界

### 8.1 本版本不做

- 社交功能（分享、评论）
- 音乐购买 / 版权保护（DRM）
- 原生移动端 App（依赖 Subsonic 生态）
- 云存储（S3 等）作为音乐库来源

### 8.2 外部依赖风险

- 网易云 / QQ 音乐 API 为非官方，存在失效风险；设计上作为可选增强，主流程不依赖
- AcoustID 和 MusicBrainz 为免费服务，需遵守请求频率限制（配置退避策略）
